# Cloudflare R2 파일 저장 설정 가이드

게시판 사진·자료실 파일을 Cloudflare R2(무료 10GB·전송 무료)에 저장하기 위한 설정입니다.
대시보드에서 클릭 몇 번이면 됩니다. (비밀키는 코드에 넣지 않습니다 — Worker '바인딩'으로 안전 연결)

## 1) R2 버킷 만들기
1. Cloudflare 대시보드 → 왼쪽 **R2 Object Storage**(스토리지 및 데이터베이스 안)
2. **Create bucket** → 이름 `church-uploads` → 생성
   - (처음이면 R2 사용 시작에 카드 등록이 필요할 수 있으나, 무료 한도 내에선 과금 안 됨)

## 2) Worker 만들기
1. 왼쪽 **Compute(컴퓨트) → Workers 및 Pages** → **Create application → Create Worker**
2. 이름 `church-files` → **Deploy**(일단 기본 코드로 배포)
3. **Edit code**(코드 편집) → 기존 내용 전부 지우고 → `cloudflare/worker.js` 내용 전체 붙여넣기 → **Deploy**

## 3) Worker에 값/버킷 연결 (Settings)
Worker 화면의 **Settings** 탭에서:

**① Variables and Secrets** (일반 변수 2개 추가)
| 이름 | 값 |
|---|---|
| `SUPABASE_URL` | `https://cetacttsdwzxjzkyozgd.supabase.co` |
| `SUPABASE_ANON_KEY` | `sb_publishable_qfq4Hvs4tF_1ZIezPoMojg_h6XNw01G` |

**② Bindings → Add → R2 bucket**
| 항목 | 값 |
|---|---|
| Variable name | `BUCKET` ← 반드시 이 이름 |
| R2 bucket | `church-uploads` |

추가 후 다시 **Deploy**.

## 4) Worker 주소 알려주기
배포되면 주소가 생깁니다: `https://church-files.<계정이름>.workers.dev`
→ 이 주소를 알려주시면 제가 `js/config.js`의 `window.R2_UPLOAD_URL`에 넣고
게시판 사진 첨부 + 자료실 R2 전환을 마무리합니다.

## 동작 요약
- 업로드: 로그인 교인만 → 브라우저에서 자동 압축 → Worker가 R2에 저장
- 보기: `…workers.dev/f/키` 로 누구나 열람(1년 캐시)
- 삭제: 본인이 올린 파일만 가능
